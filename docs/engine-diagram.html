How a lesson pack is generated from a teacher request to a finished, curriculum-aligned output

Teacher Request
Year group · Subject · Topic · Teacher profile (ability mix, SEND focus, EAL %, teaching approach) · Lesson context (unit position, AfL note, today's adjustments) · Uploaded file context
Layer 1 of 2
Cache Lookup
In-memory cache checked first (sub-ms). On cold start or miss: persistent Supabase cache checked (24h TTL, shared across all Cloud Run instances).
Cache hit → return immediately, skip all AI calls
cache miss
Curriculum Retrieval
Retrieves up to 5 UK National Curriculum objectives for this year group + subject + topic. Uses Cloudflare D1 vector search (cosine similarity on BGE embeddings) with keyword-match fallback. These objectives anchor the entire generation.

Single provider
Quality Pass
Fastest available provider reviews the draft against 11 criteria (objectives start with Bloom's verb, worked example shows all steps, greater depth demands reasoning, SEND adaptations name specific strategies, etc.). If rejected: only the flagged sections are regenerated — not the whole pack.
Single provider
Alignment Pass
Checks curriculum objective coverage ratio (≥ 60% threshold). If below threshold: fastest provider realigns objectives, teacher explanation, worked example, and activities to the retrieved NC objectives. Skipped entirely if already aligned.
Finalize
Gap-fill with sensible UK defaults · UK spelling normalisation (colour, organise, behaviour, centre, analyse) · Zod schema validation · Programmatic slide generation · Confidence scoring (high / medium / low based on objectives retrieved + context used)
Cache Write
Result written to in-memory cache immediately. Also written to Supabase engine_cache table asynchronously (non-blocking). Next request for the same topic + profile hits the cache — zero AI calls.
